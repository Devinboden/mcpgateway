# Boom AgentCore Assets тАФ read me first

A point-in-time mirror of the **Boom MCP** as deployed to **Amazon Bedrock AgentCore Runtime**, kept
in the mcpgateway repo for reference. It is a *snapshot*, not the source of truth тАФ the live source of
truth is the Boom MCP repo.

- **Source repo:** https://github.com/Noland-LAL/boom-mcp.git
- **Snapshot of:** `origin/main` @ commit **`cb2f84d`** ("Trim Boom tool surface: 11 тЖТ 5 tools").
- Application source (`src/`, `app/`, `widget/`, `vendor/`, `data/`, `package*.json`) is an exact
  extract of that commit; the `.bedrock_agentcore*` deploy files were captured from the deploying
  machine (they are not tracked in the Boom repo).

---

## Which files matter for an AgentCore deployment

The AgentCore Runtime container is tiny: it runs the streamable-HTTP MCP entrypoint and nothing else.
Everything under `app/`, `widget/`, `vendor/`, `scripts/`, and `next.config.mjs` belongs to the **Vercel /
Next.js** build and is **not used by the AgentCore runtime** (verified: `src/` imports none of it).

### тЬЕ Essential тАФ the runtime will not start without these
| File / dir | Why |
|---|---|
| `src/agentcore.js` | **The entrypoint.** Serves streamable-HTTP MCP on `0.0.0.0:8000/mcp`. `CMD` runs this. |
| `src/boomClient.js` | Boom data client (imported by the entrypoint). |
| `src/tools.js` | Registers the 5 Boom tools (imported by the entrypoint). |
| `src/ratios.js`, `src/resolve.js`, `src/intelligence.js` | Tool logic imported by `tools.js`. |
| `data/` (`atlas_spread.json`) | Fixture data. The image sets `BOOM_USE_FIXTURE=true`, so this is read at runtime. |
| `package.json` + `package-lock.json` | Dependencies. Build does `npm ci --omit=dev` against the lockfile. |

### ЁЯФз Build & deploy control files
| File | Why |
|---|---|
| **`Dockerfile.agentcore`** | **The build file to use.** ARM64, installs prod deps, copies `src/` + `data/`, `CMD node src/agentcore.js`. |
| `.bedrock_agentcore.yaml` | AgentCore toolkit deploy manifest тАФ runtime name/ARN, ECR repo, exec role, Cognito JWT authorizer, memory, region. Drives `agentcore launch`. Contains machine-specific absolute paths + account ARNs; regenerated by `agentcore configure`. |
| `.dockerignore`, `src/.dockerignore`, `.npmrc` | Build hygiene / npm config. |

### тЪая╕П Present but NOT needed for AgentCore (Vercel/Next.js only)
`app/**` (incl. `app/api/[transport]/route.js`, `app/_widgets.js`), `next.config.mjs`, `widget/**`,
`vendor/ext-apps-app-with-deps.js`, `scripts/**`, `test/**`, `LICENSE`, the Boom `README.md`.

### ЁЯЪл Do not use for AgentCore
`.bedrock_agentcore/boom_mcp/Dockerfile` тАФ this is the **toolkit-generated generic TypeScript template**.
It runs `npm run build` (which is `next build` here) and then `node dist/agentcore.js`, but this project
is plain JS with **no `dist/`** тАФ so that image would **fail**. It's included only for provenance.
**Build from `Dockerfile.agentcore` instead.**

### Minimal deployable subset
If you wanted the smallest set that deploys: `src/`, `data/`, `package.json`, `package-lock.json`,
`Dockerfile.agentcore`, and (for the toolkit) `.bedrock_agentcore.yaml`.

## How to deploy to AgentCore Runtime

**Runtime facts** (from `.bedrock_agentcore.yaml`):

| Item | Value |
|---|---|
| Runtime | `boom_mcp-fQUgkK4pjp` тАФ `arn:aws:bedrock-agentcore:us-east-1:111204669101:runtime/boom_mcp-fQUgkK4pjp` |
| ECR repo | `111204669101.dkr.ecr.us-east-1.amazonaws.com/bedrock-agentcore-boom_mcp` |
| CodeBuild | `bedrock-agentcore-boom_mcp-builder` |
| Platform / network / protocol | `linux/arm64` ┬╖ `PUBLIC` ┬╖ `MCP` (port 8000) |
| Auth | Cognito JWT authorizer, pool `us-east-1_Z1otWFj2I` |
| Memory | `boom_mcp_mem-3ugmkd30nE` (STM_ONLY, 30-day) |

**Option A тАФ AgentCore toolkit (uses `.bedrock_agentcore.yaml`).** From the Boom repo working dir:
```bash
agentcore configure --entrypoint src/agentcore.js --name boom_mcp   # regenerates the manifest for THIS machine
agentcore launch                                                     # builds via CodeBuild + updates the runtime
```
> Note: the toolkit auto-generates `.bedrock_agentcore/boom_mcp/Dockerfile`, which is wrong for this JS
> project (see above). Point the build at `Dockerfile.agentcore`, or use Option B.

**Option B тАФ build from `Dockerfile.agentcore` and roll the runtime (reliable):**
```bash
# 1. Build the ARM64 image and push to ECR (CodeBuild, or docker buildx --platform linux/arm64)
#    docker build -f Dockerfile.agentcore -t <ECR>/bedrock-agentcore-boom_mcp:vN .
#    docker push <ECR>/bedrock-agentcore-boom_mcp:vN
# 2. Point the runtime at the new image (replay the Cognito JWT authorizer + MCP protocol + PUBLIC net)
aws bedrock-agentcore-control update-agent-runtime \
  --agent-runtime-id boom_mcp-fQUgkK4pjp --region us-east-1 \
  --agent-runtime-artifact '{"containerConfiguration":{"containerUri":"<ECR>/bedrock-agentcore-boom_mcp:vN"}}' \
  --network-configuration '{"networkMode":"PUBLIC"}' \
  --protocol-configuration '{"serverProtocol":"MCP"}' \
  --authorizer-configuration '{"customJWTAuthorizer":{"discoveryUrl":"https://cognito-idp.us-east-1.amazonaws.com/us-east-1_Z1otWFj2I/.well-known/openid-configuration","allowedClients":[ ... ]}}' \
  --role-arn arn:aws:iam::111204669101:role/AmazonBedrockAgentCoreSDKRuntime-us-east-1-8b303da646
```
After it reports `READY`, re-sync the gateway target so `boom___*` tools refresh.

## Caveats
- **The live runtime image may lag this snapshot.** The deployed image was built from an earlier local
  commit; these assets reflect the *current* Boom repo (`origin/main` cb2f84d, 5 tools). Rebuild to sync.
- `.bedrock_agentcore.yaml` paths are machine-specific and regenerated by `agentcore configure`.
- To refresh this folder: `git archive origin/main` from the Boom repo, then re-overlay the
  `.bedrock_agentcore*` files.
